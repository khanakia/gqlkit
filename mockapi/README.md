# mockapi — Test GraphQL API

A mock GraphQL server built with [gqlgen](https://github.com/99designs/gqlgen) that serves as the test backend for generated SDK examples. Data is stored in-memory and seeded on first request.

## Running

```bash
go run server.go
# → GraphQL playground at http://localhost:8081/
# → Query endpoint at http://localhost:8081/query
```

Set `PORT` environment variable to change the port.

## Schema Overview

### Types

| Type | Fields |
|------|--------|
| `Todo` | `id`, `text`, `done`, `priority?`, `tags[]`, `user` |
| `User` | `id`, `name`, `email?`, `role` |
| `Role` (enum) | `ADMIN`, `USER`, `GUEST` |
| `TodoConnection` | `edges[]`, `pageInfo`, `totalCount` (Relay pagination) |
| `SearchResult` (union) | `Todo \| User` |

### Queries (9)

| Query | Description |
|-------|-------------|
| `ping` | Returns `"pong from gqlgenapi"` |
| `todo(id)` | Get single todo by ID |
| `todos(filter?, pagination?)` | List todos with filtering and offset pagination |
| `todosConnection(filter?, pagination?)` | Relay-style cursor pagination |
| `user(id)` | Get single user by ID |
| `users` | List all users |
| `search(term)` | Full-text search across todos and users (union type) |
| `echo(message)` | Echo input message back |
| `sum(a, b)` | Return sum of two integers |

### Mutations (5)

| Mutation | Description |
|----------|-------------|
| `createTodo(input)` | Create a new todo |
| `updateTodo(id, input)` | Partial update (patch semantics — nil fields are skipped) |
| `deleteTodo(id)` | Delete by ID, returns boolean |
| `upsertUser(id?, name, email?, role?)` | Create or update user |
| `completeAllTodos` | Mark all incomplete todos as done, returns count |

### Filtering (`TodoFilter`)

All filters are AND-combined:
- `textContains` — case-insensitive substring match on `text`
- `done` — exact boolean match
- `minPriority` — minimum priority threshold
- `hasTag` — case-insensitive tag match

### Pagination

- **Offset-based:** `PaginationInput { limit, offset }` (default limit=10, offset=0)
- **Cursor-based:** `TodoConnection` with `edges[].cursor`, `pageInfo { hasNextPage, hasPreviousPage, startCursor, endCursor }`

## Seed Data

On first request, the resolver seeds:
- **Users:** Alice (ADMIN), Bob (USER)
- **Todos:** "Buy groceries" (Alice, priority 3, tags: shopping/food), "Write tests" (Bob, priority 5, tags: dev)

## Package Structure

```
server.go                Main entry point — HTTP server setup
graph/
  schema.graphqls        GraphQL schema definition
  resolver.go            Resolver struct, seed data, filter helpers
  schema.resolvers.go    Query and mutation resolver implementations
  generated.go           Auto-generated by gqlgen (do not edit)
  model/
    models_gen.go        Auto-generated Go types (do not edit)
```

## Regenerating

```bash
GOWORK=off go run github.com/99designs/gqlgen@v0.17.87 generate
```

Or via Taskfile:

```bash
task mockapi:generate
```
